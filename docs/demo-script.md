# CallRevive AI — Demo Script

Follow this script to demonstrate the core capabilities of CallRevive AI in a development or staging environment.

---

## Scenario
A customer calls a local plumber (e.g., "Apex Plumbing") but the call is missed. CallRevive AI immediately intercepts the missed call, triggers an outbound AI callback, conducts a multi-turn conversation to qualify the lead, predicts a revenue recovery score, and alerts the owner.

---

## Step 1: Simulate the Missed Call Webhook

We will mock the incoming webhook that Twilio calls when a call goes unanswered.

1.  Send a POST request simulating a busy/missed call status:
    ```bash
    curl -X POST http://localhost:8000/api/v1/webhooks/twilio/voice \
      -d "CallStatus=busy" \
      -d "CallSid=CA_demo_12345" \
      -d "From=%2B1234567890" \
      -d "To=%2B1098765432"
    ```
2.  **Verify**: Check your backend logs. You should see the missed call trigger and the celery task `process_missed_call` queueing:
    ```
    worker-1  | [2026-06-09 03:00:00: INFO/MainProcess] Received task: app.worker.tasks.call_tasks.process_missed_call
    ```

---

## Step 2: The Stateful Conversation Loop (Voice AI)

Next, the customer answers the automated callback. We will simulate the multi-turn speech gather responses.

1.  **Turn 1: Customer explains their issue**:
    ```bash
    curl -X POST http://localhost:8000/api/v1/webhooks/twilio/voice/gather \
      -d "SpeechResult=My kitchen pipe is leaking and water is everywhere!" \
      -d "CallSid=CA_demo_12345" \
      -d "language=en-US"
    ```
    *   **Verify Response**: The server returns TwiML with a `<Say>` tag generated by Gemini. It will say something like: *"I'm sorry to hear that. I can schedule an emergency plumber to help you. Is that okay?"* and a `<Gather>` tag waiting for the next answer.
2.  **Turn 2: Customer agrees and requests a slot**:
    ```bash
    curl -X POST http://localhost:8000/api/v1/webhooks/twilio/voice/gather \
      -d "SpeechResult=Yes please, can you send someone tomorrow morning?" \
      -d "CallSid=CA_demo_12345" \
      -d "language=en-US"
    ```
    *   **Verify Response**: Gemini processes the confirmation, offers a time, and adds the `[END]` token to close the conversation. The server returns a goodbye TwiML response containing `<Hangup />`.

---

## Step 3: Verify Lead Qualification & Revenue Scoring

Once the call hangs up, the Celery pipeline processes the recording and transcripts.

1.  Check the SQLite/Postgres database for the newly created Lead:
    *   **Urgency**: High
    *   **Category**: Hot (due to leaking water)
    *   **Service Requested**: Emergency plumbing / leak repair
2.  Check the `RevenuePrediction` table. Gemini Pro calculates:
    *   **Conversion Probability**: ~90%
    *   **Estimated Revenue**: e.g., ₹5,000 (standard emergency plumbing fee)
    *   **Competitor Risk**: Low (customer wants immediate help)
    *   **Recommendation**: Dispatch plumber immediately.

---

## Step 4: Verify Notifications & Alert Routing

1.  Verify that an alert was dispatched to the business owner:
    *   Check backend logs for WhatsApp notification dispatches.
    *   Verify that a `hot_lead` type notification exists in the PostgreSQL `notifications` table for the registered user.
